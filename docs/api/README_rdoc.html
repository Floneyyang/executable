Executable

DESCRIPTION

The Executable mixin is a very quick and and easy way to make almost any class usable via a command line interface. It simply uses writer methods as option setters, and the first command line argument as the method to call, with the subsequent arguments passed to the method.

FEATURES

  • Super easy to use, just mixin.

  • Public writters become CLI options.

  • Public methods become subcommands.

RESORUCES

  • rubyworks.github.com/executable

  • github.com/rubyworks/executable

RELEASE NOTES

Please see HISTORY file.

SYNOPSIS

Simply mixin Executable, then call #execute_command.

class X
  include Executable

  attr_accessor :quiet

  attr_accessor :size

  def bread(*args)
    ["BREAD", quiet, size, *args]
  end

  def butter(*args)
    ["BUTTER", quiet, size, *args]
  end
end

x = X.new

x.execute_command("butter yum")
=> ["BUTTER", nil, nil, "yum"]

x.execute_command("bread --quiet --size=big")
=> ["BREAD", true, "big"]

Notice that Executable requires an equal-sign (=) be used when specifying values for non-boolean attributes.

To make the command available on the comamnd line, add an executable to your project passing ARGV to the #execute_command method.

#!usr/bin/env ruby
require 'x'
X.execute_command(ARGV)

$ gem install executable
